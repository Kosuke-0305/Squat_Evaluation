# スクワットアノテーション改善 — ClaudeCode 実装依頼

## 目的

現在のアノテーション生成はレップ境界（start_frame / end_frame）を自動検出に依存しているため、
最初のレップ数や境界が少しズレると、その後の rep_id・baseline・特徴量が連鎖的に崩れやすいです。

そのため、以下の改善を実装してください。

---

## 要件

### 1. レップ境界を大まかな精度でも人が修正できるようにする

- 既存の自動検出結果に対して、アノテーション時にレップの開始・終了位置を手動で調整できる仕組みを追加すること。
- できれば、以下のどちらかを実装すること。
  1. CLI で `--review-annotation` などのモードを追加し、候補のレップ境界を表示・修正できる
  2. `summary.json` / `annotation_candidates.csv` に `start_frame` / `end_frame` を上書き可能な形で出力し、後続の手動調整をしやすくする
- 修正は「大まかでも良い」ことを前提にし、厳密なフレーム単位ではなく、レップの開始・終了の目安を人が変更できるようにすること。

### 2. 人が動画のどの時点か分かりやすいように、時間情報を併記する

- 各レップについて、以下のような人間が見やすい時間情報を出力すること。
  - `start_time_sec`
  - `end_time_sec`
  - `deepest_time_sec`
  - 可能なら `start_mmss`, `end_mmss`, `deepest_mmss`
- これにより、動画の「何秒あたりか」をそのまま確認できるようにすること。
- フレーム番号だけではなく、`mm:ss` 形式のタイムスタンプを併記すること。

### 3. レップ判定対象から除外したい前後のルーティーンを明示的に弾けるようにする

- ラックアップ前の準備動作や、レップの前後にある無関係なルーティーンを、アノテーション対象から除外できるようにすること。
- 具体的には、次のような仕組みを検討すること。
  - `ignore_before_sec` / `ignore_after_sec`
  - `exclude_range`（開始秒〜終了秒）
  - `annotation_window`（アノテーション対象とする時間帯）
- これにより、ラックアップ前の動作や余分な動きが誤ってレップとしてカウントされないようにすること。

### 4. レップ境界の修正を、学習データ側にも反映させる

- 人が境界を修正した場合、その修正結果が次の出力に反映されるようにすること。
  - `summary.json`
  - `annotation_candidates.csv`
  - `feature_sequences.json`
- つまり、レップの境界変更が、後続の特徴量抽出やラベル付けにも影響すること。

---

## 実装方針

1. 既存の自動レップ検出ロジックを壊さず、最小変更で拡張すること。
2. レップ境界の修正は「候補の start_frame / end_frame を上書きする」形で実装すること。
3. 人が確認しやすいよう、各レップごとに次の情報を出力すること。
   - `rep_id`
   - `start_frame`, `end_frame`
   - `start_time_sec`, `end_time_sec`
   - `start_mmss`, `end_mmss`
   - `deepest_frame`, `deepest_time_sec`
4. ルーティーン除外は、レップ候補生成前にフィルタリングできるようにしておくこと。
5. 既存の `main.py` / `pose/rep_detector.py` / `README.md` に整合した形で追加すること。

---

## 追加の考え方

- 「大まかな精度でも良い」ので、厳密なフレーム単位の修正にこだわらず、
  人が動画上で見て「この辺からこの辺までが1レップ」と判断しやすい形にすること。
- 表示は、フレーム番号だけでなく秒数・mm:ss を併記することで、
  観察者が動画内のいつ頃かすぐ判断できるようにすること。
- これにより、アノテーションを人間がやりやすくしつつ、
  誤ったレップ境界が学習データにまで波及しにくくすること。

---

## 期待する完成形

- ある動画を処理すると、レップ候補に「開始時刻・終了時刻」が付く
- 人がその時間を見て大まかに修正できる
- ラックアップ前後の不要な動作はアノテーション対象から除外できる
- 修正後の境界が `summary.json` / `annotation_candidates.csv` / `feature_sequences.json` に反映される
