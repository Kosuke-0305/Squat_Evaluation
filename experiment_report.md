# スクワット有効試技判定 LSTM 実験レポート

## データ概要

| 項目 | 内容 |
|---|---|
| 総レップ数 | 361件（有効248 / 無効113） |
| 特徴量 | 11次元（膝角度・股関節角度・肩変位・足首変位等） |
| モデル | LSTM（可変長シーケンス対応） |

無効試技の失敗タイプ内訳：

| 失敗タイプ | 件数 |
|---|---|
| depth_score（深さ不足） | 46件 |
| bar_descent_score（バー下降） | 40件 |
| bounce_score（バウンス） | 10件 |
| lockout_score（ロックアウト） | 5件 |
| foot_shift_score（足移動） | 1件 |

---

## 実験の流れと結果

### Step 1：6出力LSTM（有効 + 失敗5種）

**設定**：全データ346件、多出力（valid + 5失敗スコア）

**結果**：
- モデルが全レップを「有効」と予測する自明解に収束
- val AUC=0.500、失敗要素 recall=0.000

**原因**：失敗スコアのアノテーション不足 + クラス不均衡（有効74%）

---

### Step 2：1:1バランス調整（有効を無効数にダウンサンプル）

**設定**：有効99件・無効99件の計198件

**結果**：
- val AUC=0.569（ランダム近傍）
- 依然として全部「有効」に近い予測

**原因**：問題設定が複雑すぎ、データ不足

---

### Step 3：2値分類に絞る

6出力から「有効/無効」の2値分類に問題を縮小。

**結果（バランスなし・全331件）**：
- val accuracy=0.742、AUC=0.490
- 全部「有効」と予測している状態と同等

---

### Step 4：特徴量・ルールベースの分析

**最小膝角度の比較**：

| | 平均最小膝角度 | 90°以下の割合 |
|---|---|---|
| 有効試技 | 92.9°（±16.5°） | 43.1% |
| 無効試技 | 102.7°（±17.2°） | 21.2% |

差=9.8°と一定のシグナルはあるが分布の重なりが大きい。

**ルールベース分類器の評価**：

```
accuracy=78%
無効 precision=0.97（誤検知はほぼなし）
無効 recall=0.32（68%を見逃す）
```

ルールベースの閾値（≤90°）が厳しすぎるため、軽微な深さ不足や
バー下降・バウンスによる無効を見逃していた。

**結論**：LSTMはこの時点でルールベースに勝てていなかった。

---

### Step 5：データ拡張

時系列データ拡張で無効試技を7倍に増強。

| 手法 | 倍率 | 内容 |
|---|---|---|
| time_warp | ×4 | 時系列を80%/90%/110%/120%にリサンプル |
| noise | ×2 | ガウスノイズ（角度±1.5°、変位±0.002）を加算 |
| lr_flip | ×1 | 左右対称に反転（角度交換・x変位符号反転） |

**結果**：無効試技 113件 → 904件（元113 + 拡張791）

---

### Step 6：データリーク修正・保存基準変更

**問題点の発見**：
1. 同一レップの元データと拡張データが Train/Val に混在（データリーク）
2. `pos_weight` による損失補正が val_loss を歪め、最悪モデルを保存していた（val_loss最小時にAUC=0.513）

**修正内容**：
- 検証セットを**元データのみ**に限定（64件）
- モデル保存基準を **val_loss → AUC** に変更
- スケジューラーも AUC ベースに変更（mode="max"）
- weight_decay=1e-4 を追加

---

### Step 7：5-fold 交差検証による最終評価

単一のtrain/val分割では検証セット（64件）が小さすぎるため、
5-fold CVで信頼性を確認。

**設定**：
- 元データ（331件）のみで fold 分割
- 拡張データ（791件）は各 fold の train にのみ追加
- 80エポック × 5fold

**結果**：

| Fold | 検証件数 | AUC |
|---|---|---|
| 1 | 67件 | 0.732 |
| 2 | 66件 | 0.748 |
| 3 | 66件 | 0.750 |
| 4 | 66件 | 0.711 |
| 5 | 66件 | 0.718 |
| **平均** | | **0.732 ± 0.016** |

---

## 全手法の比較

| 手法 | AUC（推定） | 無効 Recall |
|---|---|---|
| 全部「有効」と予測 | 0.500 | 0% |
| ルールベース | ~0.657 | 32% |
| 6出力LSTM | ~0.500 | ~0% |
| 2値LSTM（拡張なし） | ~0.500 | ~0% |
| **2値LSTM（拡張あり・5fold CV）** | **0.732 ± 0.016** | **~53%** |

---

## 考察

### 有効だった施策
- **2値分類への絞り込み**：失敗スコアのアノテーション不足問題を回避
- **データ拡張**：無効試技の実効サンプル数を8倍に増加
- **保存基準のAUC化**：pos_weightによる損失の歪みを回避
- **5-fold CV**：小さな検証セット（64件）の統計的ノイズを軽減

### 残存する課題
- **bounce/lockout/foot_shift** の件数不足（10件・5件・1件）により、これらの失敗タイプはほぼ学習できていない
- 現時点の AUC=0.732 は **depth と bar_descent の識別** がほとんどを担っている可能性が高い

### さらなる改善に必要なこと
各失敗タイプごとに **100件以上** の実データ収集が必要。
特に再現しやすい bounce・lockout・foot_shift を優先的に収集することで、
AUC=0.8 以上が期待できる。
